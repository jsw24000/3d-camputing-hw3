## colmap_sparse_geometry

- Role: Traditional geometry baseline and shared pose source.
- Representation: Sparse point cloud plus calibrated pinhole/radial camera model.
- Architecture / optimizer: Feature extraction + exhaustive matching + incremental SfM + bundle adjustment.
- Training objective: Minimize multi-view reprojection error over camera poses, intrinsics, and 3D points.
- Adapter script: `scripts/run_colmap.py`
- Status: completed for current video

Inputs:
- `data/images`
- `data/images_train`

Outputs:
- `results/colmap/sparse_point_cloud.ply`
- `results/colmap/train_sparse_point_cloud.ply`
- `results/colmap/train_geometry_summary.json`
- `results/colmap/train_geometry_preview.png`

Learned or estimated parameters:
- camera poses
- single shared SIMPLE_RADIAL intrinsics
- 3D sparse point coordinates
- track associations and reprojection errors

## nerfstudio_nerfacto

- Role: Neural radiance-field view-synthesis method.
- Representation: Continuous neural field queried by 3D position and view direction; renders RGB/density along camera rays.
- Architecture / optimizer: Nerfstudio Nerfacto implementation; project adapter supplies COLMAP-derived transforms and timing/eval wrappers.
- Training objective: Photometric reconstruction of training views with Nerfacto losses and ray sampling.
- Adapter script: `scripts/run_nerfstudio.py`
- Status: data ready; external nerfstudio dependency not installed yet

Inputs:
- `data/nerfstudio_colmap_train/transforms.json`
- `data/images_train`

Hold-out / evaluation input:
- `data/nerfstudio_colmap_test/transforms.json`
- `data/images_test`

Outputs:
- `results/nerfstudio`
- `results/nerfstudio/eval.json`
- `results/nerfstudio/eval_renders`

Learned or estimated parameters:
- Nerfacto field/network weights
- proposal sampling components
- appearance/radiance parameters

Train Command:

```powershell
conda run -n scene-nerf python scripts/run_experiment_suite.py --method nerfstudio --stage train --env-mode fallback
```

Eval Command:

```powershell
conda run -n scene-nerf python scripts/run_experiment_suite.py --method nerfstudio --stage eval --env-mode fallback
```

## official_3dgs

- Role: Explicit radiance primitive representation.
- Representation: Set of anisotropic 3D Gaussian primitives with position, covariance/scale/rotation, opacity, and spherical-harmonic color coefficients.
- Architecture / optimizer: Official GraphDeCo 3D Gaussian Splatting training loop initialized from COLMAP sparse points.
- Training objective: Differentiable Gaussian splatting render loss against training images, with densification/pruning during optimization.
- Adapter script: `scripts/run_3dgs.py`
- Status: completed for current video

Inputs:
- `data/3dgs_train`
- `data/3dgs_train`

Hold-out / evaluation input:
- `data/3dgs`
- `data/3dgs`

Outputs:
- `results/3dgs/eval_model`
- `results/3dgs/eval_model/test/ours_30000/renders`
- `results/3dgs/eval_model/test/ours_30000/gt`
- `results/3dgs/full_model/train/ours_30000/renders`
- `results/3dgs/full_model/point_cloud/iteration_30000/point_cloud.ply`

Learned or estimated parameters:
- Gaussian means
- Gaussian covariance / scale / rotation
- opacity
- color / spherical harmonic coefficients

Train Command:

```powershell
conda run -n scene-3dgs python scripts/run_experiment_suite.py --method 3dgs --stage train --env-mode fallback
```

Render Command:

```powershell
conda run -n scene-3dgs python scripts/run_experiment_suite.py --method 3dgs --stage render --env-mode fallback
```

## vggt_gs

- Role: Extra comparison using pretrained feed-forward geometry before GS training.
- Representation: VGGT-predicted cameras/point maps converted to COLMAP-style geometry, then optimized with a GS/gsplat path.
- Architecture / optimizer: Facebook VGGT export pipeline plus gsplat/simple trainer adapter.
- Training objective: Use pretrained VGGT predictions as geometry initialization, then optimize Gaussian rendering if gsplat trainer is available.
- Adapter script: `scripts/run_vggt_gs.py`
- Status: entry ready; VGGT and gsplat dependencies not installed yet

Inputs:
- `data/images_train`
- `data/vggt_scene_train`

Outputs:
- `results/vggt_gs`
- `results/vggt_gs/renders`

Learned or estimated parameters:
- VGGT-predicted camera parameters
- VGGT point/depth maps
- downstream Gaussian parameters

Train Command:

```powershell
conda run -n scene-vggt python scripts/run_experiment_suite.py --method vggt_gs --stage train --env-mode fallback
```
